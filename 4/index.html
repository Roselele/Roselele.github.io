A.2: Recover Homographies

In this subpart, we compute the transformation matrix, i.e. the homography matrix with 8 DoF, by constructing and solving a linear system of equations from a set of corresponding points.

At least four corresponding points are required to recover the homography matrix because each point provides x and y coordinate information which is 2 DoF and we need to solve 8 unknowns in total.

However when we implement the recovery in practice, we find that even some small deviation of the coordinates can leads to huge error when the matrix is uniquely solved from the 4 corresponding points. The warping and mosaic result appears to be rather unsatisfying.

correspondence with 4 points
unsatisfying result

We can see the mappings are good but the mosaic looks terrible.

So on the second trial, I selected 8 points to recover a better estimate of the projection transformation. The linear equation solver will return the least square result in this case which gives a much more robust solution. And the warping and mosaic created by this new homography matrix is much better.

Methodology

Step 1: Use matplotlib tool to manually choose a set of corresponding points on two images.

Step 2: Create the linear equation λ[u,v,1] = H[x,y,1], extend the matrix to equations and simplify by eliminating λ, now we get 2 equations for each corresponding point containing the coordinate of the points.

for each pair of points (x, y) -> (x_p, y_p), we can construct:

[x, y, 1, 0, 0, 0, -x*x_p, -y*x_p, -x_p] H^T = 0
[0, 0, 0, x, y, 1, -x*y_p, -y*y_p, -y_p] H^T = 0

Step 3: Use linear equation solver to get the terms in the matrix.

A.3: Warp the Images

In this subpart, we implement inverse warping with both nearest neighbor and bilinear interpolation methods.
In order to get the pixel colors of the warped image, we need to sample from the source image. We first get the boundary of the new image by getting the four new corners, then for each pixel in the new image, we use inverse transformation to find where the pixel is in the original image. In nearest-neighbor method, we simply use the color of the nearest pixel at that coordinate as the color of the new pixel. In bilinear method, we use a weighted average of the 4 nearby pixels to acquire the exact color in the new image.
Below present the basic idea of how we sample the pixels

#rounding to nearest neighbor x_nn = int(round(x_src)) y_nn = int(round(y_src))
x0, y0 = int(np.floor(x_src)), int(np.floor(y_src)) x1, y1 = x0 + 1, y0 + 1 wx = x_src - x0 wy = y_src - y0 interpolated = (1 - wx) * (1 - wy) * im[y0, x0] + \ wx * (1 - wy) * im[y0, x1] + \ (1 - wx) * wy * im[y1, x0] + \ wx * wy * im[y1, x1]

Project 3B: Feature Matching for Autostitching

Implementing automatic feature detection, matching, and robust homography estimation for image mosaicing

B.1: Harris Corner Detection

In this section, we implement the Harris corner detector to identify interest points in images. The Harris detector measures the local changes in image intensity to find corners, which are locations where intensity changes significantly in multiple directions.

Methodology: Used the provided Harris corner detection code to identify corners. Implemented Adaptive Non-Maximal Suppression (ANMS) to select the most distinctive corners while ensuring good spatial distribution across the image.

Harris Corners Without ANMS

Image 1 - All Harris Corners
Image 2 - All Harris Corners

We can see Harris corner detector is finding hundreds or thousands of "interest points". If we simply take the top N strongest points (based on their "corner strength" score), they tend to cluster in the most textured areas of the image. This is bad for tasks like image stitching because:

Computational Cost:
Matching is computationally expensive, and you want to limit the number of points.
Poor Coverage:
If all your points are in one small, highly textured region, you might have no points in the overlapping area of two images, causing the matching to fail.

Therefore we need ANMS! ANMS solves the above problem by selecting a set of points that are not only strong but also well-spread across the entire image.

How ANMS works

The key idea is to favor points that are the "strongest in their local neighborhood. A point in a relatively flat region of the image might have a medium strength score, but if it's the strongest point for a large area around it, it's very important and should be kept. Conversely, a point in a busy texture might have a very high score, but if it's surrounded by other high-scoring points, many of them are redundant.

ANMS formalizes this idea by assigning each point a "suppression radius" (r_i). A large r_i means you have to look far away to find another point that is significantly stronger, making this point a good, isolated candidate.

B.2: Feature Descriptor Extraction

For each detected corner, we extract a feature descriptor that captures the local appearance around the point. We use 8×8 patches sampled from a larger 40×40 window around each feature point.

Methodology: For each Harris corner, extract a 40×40 patch centered at the corner. Sample an 8×8 descriptor from this patch after applying appropriate blurring. Apply bias/gain normalization to make descriptors invariant to lighting changes.

B.3: Feature Matching

We match feature descriptors between two images using Lowe's ratio test. This method compares the distance to the nearest neighbor versus the second nearest neighbor to ensure distinctive matches.

Methodology: For each descriptor in the first image, find the two closest descriptors in the second image using Euclidean distance. Apply Lowe's ratio test with threshold 0.8 to filter ambiguous matches.

B.4: RANSAC for Robust Homography

We use RANSAC (RANdom SAmple Consensus) to compute a robust homography estimate from potentially noisy feature matches. This method iteratively selects random subsets of points to find the homography that has the most inliers.

Methodology: Implemented 4-point RANSAC with the following parameters: 1000 iterations, inlier threshold of 5 pixels. For each iteration, randomly select 4 correspondences, compute homography, count inliers, and keep the homography with maximum inliers.

RANSAC Implementation Details

RANSAC Algorithm:

  1. Randomly select 4 feature matches
  2. Compute homography H from these 4 points
  3. Project all points from image 1 to image 2 using H
  4. Count inliers (points within threshold distance)
  5. Repeat for N iterations, keep H with most inliers
  6. Recompute H using all inliers

Conclusion

The automatic stitching pipeline successfully demonstrates the complete process from feature detection to final mosaic creation. Key insights:

  • ANMS significantly improves the spatial distribution of features compared to raw Harris corners
  • Lowe's ratio test effectively filters ambiguous matches, improving matching accuracy
  • RANSAC provides robust homography estimation even with significant outlier contamination
  • Automatic results are comparable to manual stitching while being significantly faster and more reproducible

The implemented system can reliably create high-quality mosaics from image pairs with sufficient overlap and distinctive features.
